What it takes to send data between PC and Server:
-------------------------------------------------
Step 1: The operating system network stack enacapsulate the application and transport layer data and builds an IP packet with the sourse address 192.168.0.101
Step 2: Encapsulate IP data as an Ethernet frame
Step 3: ARP (address resolution protocol) response - asks all nodes for the MAC address of the destination system
Step 4: Switch forewards the frame to the destination

Switches work at Layer 2 of the OSI model (Data Link layer) and use MAC addresses to forward Ethernet frames within the same local network (e.g., between PC1, PC2, and PC3 in Network 1 via Switch 1)

Routers operate at Layer 3 (Network layer) and use IP addresses to route packets between different networks (e.g., from PC1 in Network 1 to PC4 in Network 2).

1. Local Delivery Within a Network (Switch Role):
If traffic stays in the same subnet (e.g., PC1 to PC3, both 192.168.0.x):
- PC1 sends an Ethernet frame with PC3's MAC as the destination (learned via ARP).
- Switch 1 checks its MAC address table: "PC3's MAC is on port 3" â†’ forwards the frame out that port. No IP routing needed.
- This is fast and doesn't leave the local network.

2. Inter-Network Delivery (Router Role):
If traffic crosses subnets (e.g., PC1 to PC4):
- PC1 sees PC4's IP (200.0.1.45) is not local, so it sends the IP packet to its default gateway (the router's 192.168.0.1 interface, learned via DHCP or config). The frame uses the router's MAC as destination.
- Switch 1 forwards to the router based on the router's MAC (from its table).
- Router receives the packet, strips the Layer 2 frame, checks the destination IP (200.0.1.45) in its routing table (e.g., "send 200.0.1.0/24 via interface to Switch 2").
- Router re-encapsulates the IP packet in a new Ethernet frame (now using Switch 2's MAC as next-hop destination).
- Switch 2 receives it and forwards to PC4 based on PC4's MAC (learned via ARP on Network 2).
